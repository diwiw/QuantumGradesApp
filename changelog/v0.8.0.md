# Changelog — QuantGradesApp

---

# v0.8.0 — CLI Overhaul + Config I/O + E2E Tests

### Added
- New `input.path` / `output.path` configuration fields in JSON.
- CLI now supports:
  - `--config <file>` (required)
  - `--input <file>` (optional override)
  - `--output <file>` (optional override)
- E2E test suite:
  - Happy path: config + CSV → backtest → export
  - Negative path: missing CSV → error handling
- Full CLI logging using `LoggerFactory::createConsoleLogger`.
- DataExporter integration in CLI (CSV mode).
- Deterministic fixture set under `tests/fixtures/e2e/`.

### Changed
- Config JSON loader extended to parse I/O paths.
- Correct precedence: **CLI > JSON > defaults**.
- AppCLI fully rewritten to clean, deterministic workflow.
- Error messages standardized (`ERROR:` prefix).
- Replaced `fmt::format` usage for `std::filesystem::path` → use `.string()`.
- `Config.cpp` updated to avoid ambiguous JSON → path assignment.

### Fixed
- Input/output paths correctly loaded from JSON.
- E2E path resolution through deterministic project root search.
- Output file checks improved (size + contents).
- Logger now reinitializes properly on reload.
- Negative tests detect error messages reliably.

### Removed
- Legacy CLI logic not supporting ingest/export routing.
- Old config handling without dedicated input/output sections.

### Notes
- Windows builds now **require vcpkg**.
  Instructions added to README (Windows prerequisites).
- CI/CD temporarily limited until full test cleanup for v1.0.0.

# v0.7.1 — Infra + Validation + Logger + Demo Refactor

### Added
- DataExporter class for saving BarSeries to CSV.
- HTTP ingest (initial version).
- Improved validation of CSV rows.
- Unified Result model shared by all strategies.
- Added CLI11 integration (first version).
- Logging refactor via LoggerFactory, NullLogger, MockLogger.

### Changed
- Engine result API unified.
- Strategy selection and error model cleaned up.
- Build system improvements (CMake v0.7.x upgrade).

### Fixed
- Curl linking issues on Linux.
- Initial Windows/MSVC compatibility via vcpkg.
